# Contributing to CloudSIP

Thank you for helping improve CloudSIP.

## Workflow

1. Fork the repository.
2. Create a focused branch for your change.
3. Make and test your update.
4. Keep the project standalone and static.
5. Open a pull request with a clear summary.

## Project constraints

CloudSIP is intentionally upload-and-use:

- Do not add npm, `package.json`, package-lock files, bundlers, transpilers, or build tools unless the change has been discussed and approved first.
- Do not require a backend for default operation.
- Keep source files readable and easy to audit.
- Keep deployment compatible with static hosting.

## UI guidelines

- Keep UI behavior and styling consistent with the existing application.
- Prefer small, focused changes.
- Avoid unrelated formatting churn.
- Preserve accessibility and keyboard usability.

## Pull requests

Include:

- What changed
- Why it changed
- How it was tested
- Any browser or SIP server configuration notes
